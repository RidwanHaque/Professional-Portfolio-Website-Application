## Retaining context for your codebase "long-term"

You can't make the model's context window infinite, but you can externalize your project's context so future chats stay grounded:

- **Living design docs in the repo**: Maintain a concise `ARCHITECTURE.md` plus smaller docs per subsystem (e.g., "Placement pipeline overview," "Robot motion controller invariants"). Copilot can read these files on demand, and you can re-paste relevant sections into new threads.
- High-signal comments and docstrings: Put key assumptions, data formats, and interfaces in code comments and header files so any new chat can re-learn them just by opening the file.
- Use project-wide search instead of conversation memory: When you need past reasoning, search your repo or notes (e.g., `git grep`, Obsidian, Notion) and paste only the minimal relevant chunk into the current chat.
- Snapshot important guidance: When you get a really good explanation or plan from Copilot, paste it into a `notes/` directory in the repo or into an issue, so you can reintroduce it in later chats without dragging the whole history. [perplexity](https://www.perplexity.ai/search/371c17e7-64cd-49ab-9dfd-82f76af9bd2f)

### Example structure that scales

| Artifact type | Purpose | How it helps with context |
| --- | --- | --- |
| `ARCHITECTURE.md` | High-level system layout and data flow | Lets any new chat pick up core design in a few hundred tokens. |
| `docs/module_name.md` | Details and invariants per subsystem | You can paste just the relevant module doc when asking a question. |
| `notes/ai-sessions.md` | Curated best answers from AI tools | Preserves the useful bits of long chats without keeping the whole thread. [perplexity](https://www.perplexity.ai/search/371c17e7-64cd-49ab-9dfd-82f76af9bd2f) |
| Issues/PR descriptions | Rationale for big changes | Future chats can see why by reading a short diff and description. |

For your SAPR and robot projects, this kind of lightweight documentation will give you a reusable "memory" that survives far beyond any single Copilot chat.
